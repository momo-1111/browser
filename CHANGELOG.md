# Changelog

## 2026-04-19


- 修复阅读器模式与朗读模式的联动触发与初始化时序问题，恢复主要网页场景的内容识别与朗读流程。
- 调整阅读器模式展示逻辑，支持网页保活叠层阅读，并完善空态/加载态显示。
- 调整朗读控制栏为页面悬浮层，显示在网页与底部工具栏上方。
- 移除阅读器页面中的调试信息展示与调试串回填，恢复面向用户的正式界面。

### Bug 产生原因

- 状态同步不一致：设置弹窗与主页曾使用不同状态源，导致“已开启阅读器”与页面实际状态不同步，触发链路丢失。
- 初始化时序竞态：页面生命周期触发识别时，`TtsReaderService` 在部分路径下尚未初始化，识别流程提前返回，表现为无正文或无后续动作。
- 布局层级问题：朗读控制条最初位于普通底部流式布局中，被网页/工具栏处于同层竞争，造成“已开启但视觉上不在最上层”的体验问题。
