该方案默认采用“自动识别优先，手动入口兜底”的阅读器策略，以及“系统内置离线优先”的朗读策略；不新增顶层目录，仅在现有模块目录下增加必要能力文件。

### 第一部分：软件需求规格说明书 (SRS) 草稿

**1. 引言**
- 目的：在现有单页浏览与历史记录能力基础上，新增“阅读器模式 + 朗读功能”，提升长文阅读效率与可访问性。
- 系统状态：当前系统已具备网页加载、地址栏搜索、页面前进后退、历史记录入库、设置弹窗等能力，核心入口位于 entry/src/main/ets/pages/Index.ets。

**2. 功能需求**

| 模块 | 功能ID | 功能名称 | 需求描述 (基于代码现状) |
| :--- | :--- | :--- | :--- |
| 阅读器模式 | RD-01 | 可用性识别与触发 | 页面加载完成后进行文章页识别并输出可读性分值；采用“自动识别优先 + 手动入口兜底”，识别失败不影响原网页显示。 |
| 阅读器模式 | RD-02 | 正文提取与净化 | 提取标题、段落、图片、发布时间等正文信息，过滤导航、广告、推荐流、脚本样式等噪音节点，形成结构化内容。 |
| 阅读器模式 | RD-03 | 阅读器视图与回退 | 进入阅读器后展示纯净排版，支持字体大小、行高、主题调整；置信度低或解析异常时可提示并安全回退原网页。 |
| 朗读功能 | TTS-01 | 朗读数据源与离线优先 | 朗读仅基于阅读器结构化正文，优先使用系统内置 TTS，确保无网络时核心播报可用。 |
| 朗读功能 | TTS-02 | 播放控制与进度同步 | 提供播放、暂停、继续、停止、段落跳转、语速调节；播放进度与当前段落保持同步。 |
| 朗读功能 | TTS-03 | 异常恢复机制 | TTS 初始化失败、段落为空、页面切换中断等场景下提供明确提示、重试与恢复能力。 |
| 数据与设置 | CFG-01 | 偏好持久化与隐私兼容 | 保存阅读器与朗读偏好（字体、主题、语速、自动朗读）；隐私模式下不落盘正文缓存与朗读历史。 |
| 性能与稳定 | NFR-01 | 交互与内存约束 | 阅读器进入耗时可控，长文滚动流畅；正文解析与缓存采用分段策略，避免主线程卡顿与内存峰值过高。 |

**3. 验收标准**
1. 阅读器识别与触发：页面加载后可给出可用性判断；自动识别失败时用户仍可手动触发，且不影响原网页浏览。
2. 阅读器内容与回退：可稳定提取正文并完成纯净渲染；解析异常或置信度低时可提示并一键回退到原网页。
3. 朗读核心能力：基于阅读器结构化正文完成播报，支持播放/暂停/继续/停止、段落跳转、语速调节，并保持进度同步。
4. 隐私与设置：普通模式下偏好可持久化；隐私模式下不落盘正文缓存与朗读历史。
5. 性能与回归：长文场景下滚动与交互无明显卡顿，且搜索、前进后退、历史记录、设置弹窗等既有能力无回归。

### 第二部分：软件设计文档 (SDD) 

**1. 架构设计 (Architecture)**
- 模式：在现有页面架构上引入轻量 Service + 状态管理分层，保持页面入口不变。
- 核心变更：从“仅渲染原网页”扩展为“网页渲染 + 可读内容抽取 + 阅读器渲染 + 朗读控制”的并行能力；不调整项目顶层结构。

**2. 详细设计**

#### A. 数据模型层 
- 建议在现有目录内新增模型文件（例如放在 common 或新增 model 子目录，均位于 entry/src/main/ets 下，不新增顶层目录）。
- 结构化对象建议：
1. ArticleContent
- url: 原始地址
- title: 标题
- byline: 作者（可空）
- publishTime: 发布时间（可空）
- paragraphs: 段落数组
- images: 图片数组（可选）
- confidence: 提取置信度
- extractedAt: 提取时间

2. ReaderPreferences
- fontScale
- lineHeight
- themeMode
- autoOpenWhenAvailable

3. TtsPreferences
- speed
- pitch
- autoPlayOnReaderOpen
- resumeFromLastParagraph

4. TtsSessionState
- isPlaying
- currentParagraphIndex
- currentSentenceIndex
- progressPercent
- lastError

#### B. 业务逻辑层 (ViewModel / Service)
- ReaderExtractionService（建议放在 utils 或 service 目录）
1. 输入：当前页面 HTML 或运行时 DOM 快照。
2. 处理：节点打分、正文候选聚合、噪音过滤、置信度评估。
3. 输出：ArticleContent 或失败原因。

- ReaderModeManager
1. 管理阅读器开关状态、可用性检测状态、解析任务状态。
2. 对接页面层：控制原网页与阅读器视图切换。
3. 对接偏好层：读取和更新阅读器设置。

- TtsOrchestrator
1. 接收 ArticleContent 段落，构建播报队列。
2. 管理播放状态机：Idle -> Preparing -> Playing -> Paused -> Completed / Error。
3. 处理段落跳转、重试恢复、页面退出清理。

- PersistenceAdapter（可复用现有持久化风格）
1. 普通模式：写入阅读器偏好和朗读偏好。
2. 隐私模式：仅内存态，不落盘。

#### C. 视图层 (View) - 重构重点
- 现有文件改造建议：
1. entry/src/main/ets/pages/Index.ets
- 增加阅读器可用标识、阅读器开关状态、解析加载态、错误提示态。
- 在 Web 页面上层增加阅读器容器切换。
- 页面结束加载后触发自动识别任务（异步，失败不阻塞）。

2. entry/src/main/ets/common/bean/FunctionButtons.ets
- 增加阅读器入口按钮与朗读入口按钮（或在阅读器内部提供朗读控制条）。
- 按页面状态动态启用或禁用按钮。

3. entry/src/main/ets/view/SettingDialog.ets
- 增加阅读器和朗读偏好项：字体、主题、语速、自动朗读等。

- 新增视图建议（不改顶层结构）：
1. ReaderPage 或 ReaderPanel 组件
- 渲染 ArticleContent
- 承载字体/主题控制
- 提供“返回原网页”入口

2. TtsControlBar 组件
- 展示播放控制与进度
- 支持段落跳转与语速调节

#### D. 关键算法策略（阅读器）
1. 候选节点筛选
- 优先考虑 article、main、section、p、h1-h3、blockquote、li。
- 排除 nav、header、footer、aside、script、style。

2. 分值计算
- 文本长度、标点密度、段落连续性、链接占比、容器深度。
- 结合标题匹配度与主内容区位置，输出最终 confidence。

3. 兜底策略
- confidence 高：自动提示可进入阅读器并允许自动打开。
- confidence 中：仅展示手动入口。
- confidence 低：提示不适配并保持原网页模式。

#### E. 朗读状态流（核心）
1. 初始化：校验正文可读与段落非空。
2. 准备：按段落拆句，建立播放队列。
3. 播放：逐句推进，记录 paragraphIndex 与 progress。
4. 暂停/恢复：保持索引与偏移，恢复时继续。
5. 结束：状态归档，可重新播放。
6. 异常：记录错误码，允许重试与切换段落继续。

| Web 内容抽取 | 需实现 DOM 解析、正文打分、噪音过滤 | 需补充：当前以网页展示为主，尚无阅读器抽取模块 |
| TTS 控制 | 需接入系统朗读能力并维护播放状态机 | 需补充：当前无朗读控制层 |
| 设置持久化 | 需复用现有持久化机制保存偏好 | 良好：已有持久化与数据库工具可参考 |
| 隐私模式兼容 | 需区分落盘策略，避免隐私泄漏 | 良好：已有隐私模式状态，可扩展策略分支 |
| 性能与内存 | 长文分段解析与按需渲染，避免卡顿 | 中等：需新增分段与懒处理逻辑 |

#### 2. 工作量与难度拆解 (Workload Analysis)

| 子模块 | 难度 | 预估工时 | 关键动作与风险 |
| :--- | :--- | :--- | :--- |
| 阅读器抽取引擎 | 高 | 2-3 天 | 建立打分与过滤规则；风险在于站点差异大导致误判，需分级兜底。 |
| 阅读器视图与样式 | 中 | 1-1.5 天 | 新增纯净阅读视图并联动设置；风险在于与原 Web 视图切换时状态同步。 |
| 自动识别与入口整合 | 中 | 1 天 | 页面加载后异步识别并更新入口状态；风险在于识别耗时影响交互。 |
| 朗读控制层 | 中高 | 1.5-2 天 | 实现播放状态机、段落跳转、异常重试；风险在边界状态切换。 |
| 设置与隐私兼容 | 中 | 0.5-1 天 | 扩展偏好项并区分隐私模式落盘；风险在状态分支遗漏。 |
| 联调与回归测试 | 中 | 1-1.5 天 | 覆盖阅读器/朗读/原网页共存场景；风险在历史功能回归。 |

#### 3. 风险与建议 (Risk Management)
1. 提取误判风险
- 原因：不同站点 DOM 结构差异大。
- 建议：采用置信度分级与失败回退，默认不强制覆盖原网页。

2. 播放状态错乱风险
- 原因：暂停、跳段、页面切换并发触发。
- 建议：统一状态机入口，所有控制命令串行化处理。

3. 长文性能风险
- 原因：一次性处理全文导致主线程压力。
- 建议：段落分批处理、按需渲染、控制最大缓存窗口。

4. 隐私数据泄露风险
- 原因：隐私模式下误写偏好或正文缓存。
- 建议：持久化层统一网关判断，隐私态强制内存策略。

5. 旧功能回归风险
- 原因：入口与页面状态扩展影响原导航逻辑。
- 建议：建立回归清单，优先验证搜索、前进后退、历史写入、设置弹窗。
